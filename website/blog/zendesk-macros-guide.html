1. What Are Zendesk Macros?

A Zendesk macro is a prepared set of actions that agents can apply to a ticket with one click. At their simplest, macros insert a pre-written response into the reply field. At their most powerful, they combine a canned response with automatic ticket field updates, such as changing the status, priority, assignee group, or tags, all in a single action.

Think of Zendesk macros as canned responses with superpowers. A regular canned response just inserts text. A Zendesk macro can insert text, set the ticket to "Pending," assign it to the billing team, add a "billing-inquiry" tag, and change the priority to "High," all simultaneously. One click replaces what would otherwise be five or six separate manual steps.

Quick example: An agent receives a password reset request. Instead of typing out instructions, manually changing the ticket category to "Account Access," and setting the status to "Pending," they apply the Password Reset macro. The response, status change, categorization, and tag assignment all happen instantly.

Zendesk offers two types of macros. Personal macros are created by individual agents and only visible to that agent. Shared macros are created by administrators and available to the entire team or specific groups. For most teams, shared macros are where the real value lives because they standardize how the entire organization communicates.

Macros support Zendesk's built-in placeholders, which are dynamic variables that auto-populate with ticket data. For example, {{ticket.requester.first_name}} inserts the customer's first name, and {{ticket.id}} inserts the ticket number. These placeholders make your macros feel personal even though they are templated.

If you are coming from other support tools, Zendesk macros are similar to what Freshdesk calls "canned responses," what Intercom calls "saved replies," or what Help Scout calls "saved replies." The terminology differs but the concept is the same: pre-built responses that save time and enforce consistency. 2. How to Create Zendesk Macros (Step-by-Step)

Creating a macro in Zendesk takes less than two minutes. Here is the process for both personal and shared macros.

Creating a Personal Macro

Personal macros are great for agents who have their own frequently used responses that might not apply to the whole team. Any agent can create personal macros without admin access.

  1. Open any ticket in your Zendesk agent workspace. You need an active ticket to access the macro creation interface from the ticket view.
  2. Click "Apply macro" at the bottom of the ticket sidebar, then select "Create macro from current ticket" if you want to base it on your current reply, or navigate to your profile and select "Macros" from the menu.
  3. Give it a descriptive name. Use a naming convention that makes the macro easy to find. For example, "Account - Password Reset Instructions" is better than "reset." We will cover naming best practices in the pro tips section.
  4. Define the actions. At minimum, add a "Comment/description" action and type or paste your response text. You can also add actions for setting the status, priority, type, group, assignee, or tags.
  5. Insert placeholders. Click the placeholder button in the text editor to browse available dynamic content. Common ones include {{ticket.requester.first_name}}, {{ticket.id}}, and {{ticket.url}}.
  6. Save the macro. Click "Create" to save. Your macro is now available from the "Apply macro" menu on any ticket.

Creating a Shared Macro (Admin Required)

Shared macros are available to all agents or to specific groups. You need admin or manager permissions to create them.

  1. Go to Admin Center. Click the gear icon in the Zendesk sidebar, then navigate to "Workspaces" and select "Agent tools," then "Macros."
  2. Click "Add macro." This opens the macro builder with the same interface as personal macros, but with additional visibility options.
  3. Set visibility. Choose whether the macro is available to all agents or only to specific groups. For example, a billing-specific macro might only be visible to your billing team.
  4. Build your actions. Add the comment text, ticket field changes, and any other automated actions. Test each placeholder to make sure it resolves correctly.
  5. Save and communicate. Save the macro, then let your team know it exists. The best macro in the world is useless if agents do not know about it.
Pro tip: Before creating a macro from scratch, check if Zendesk already offers a similar one in their suggested macros. Zendesk pre-populates some starter macros for new accounts, and you can customize these rather than building from zero.

Once your macros are set up, agents can apply them from the macro menu at the bottom of any ticket. Zendesk also supports searching for macros by name, which is helpful when your library grows past a dozen templates. 5. Pro Tips for Zendesk Macro Management

A small library of five macros is easy to manage. But as your team grows and your product evolves, macro libraries balloon quickly. Without structure, agents waste time searching for the right macro, use outdated ones, or create duplicates. Here is how to keep your macro library clean and effective.

Use a Consistent Naming Convention

The number one macro management mistake is inconsistent naming. When one macro is called "password reset," another is "Reset password for customer," and a third is "PW reset v2," nobody can find anything. Adopt a naming convention and enforce it across the team.

The best format we have seen is: Category :: Action - Detail. For example:

  • Account :: Password Reset - Standard Instructions
  • Billing :: Refund - Full Refund Processed
  • Triage :: Escalate - High Priority Engineering
  • Closing :: No Response - 7 Day Follow-Up

The double colon separates the category from the action, making it easy to scan. When agents type in the macro search bar, they can search by category or by action. This convention scales to hundreds of macros without becoming chaotic.

Organize with Macro Categories

Zendesk allows you to restrict macro visibility by group. Take advantage of this. Your billing team should not see engineering escalation macros, and your technical support agents do not need sales outreach templates. Restricting visibility by group reduces clutter and ensures agents see only the macros relevant to their role.

If you need more granular organization than Zendesk's native grouping provides, consider maintaining a shared document or wiki that lists all macros by category with descriptions. This doubles as training documentation for new hires.

Maintenance metric: Teams with more than 50 macros should audit their library at least once per quarter. Our research suggests that 15-25% of macros in the average Zendesk instance are either duplicated, outdated, or unused. A quarterly cleanup keeps response quality high and prevents agents from accidentally using stale information.

Leverage Zendesk Placeholders Effectively

Zendesk offers dozens of dynamic placeholders beyond just the customer name and ticket ID. Here are some of the most useful ones that teams often overlook:

  • {{ticket.requester.first_name}} - Customer's first name for a personal touch
  • {{ticket.id}} - Ticket number for reference in communications
  • {{current_user.first_name}} - Agent's name for the signature
  • {{ticket.created_at}} - When the ticket was submitted
  • {{ticket.organization.name}} - The customer's company name
  • {{ticket.via}} - The channel the ticket came through (email, chat, etc.)

The more placeholders you use, the more personalized your macros feel without any extra effort from the agent. A response that opens with the customer's name and references their company feels tailored even though it is completely automated.

Set a Review and Retirement Cadence

Macros are not a set-it-and-forget-it tool. Policies change, product features ship, pricing updates happen. A macro that was accurate three months ago might now contain wrong information about your return policy or reference a feature that no longer exists.

Build a review cadence into your team's workflow:

  • Monthly: Review the top 10 most-used macros for accuracy. These are your highest-impact templates and any errors in them will affect the most customers.
  • Quarterly: Audit the entire macro library. Delete unused macros, update policy references, consolidate duplicates, and check that all placeholders still resolve correctly.
  • After every product launch: Create new macros for the new feature and update any existing macros that reference affected functionality.
Pro tip: Assign macro ownership. Each macro should have an owner, the person responsible for keeping it accurate. When a policy changes, the owner updates the relevant macros. Without ownership, macros slowly rot and nobody takes responsibility for fixing them.

Track Macro Usage and Performance

Zendesk's reporting tools let you see which macros are used most frequently and by which agents. Use this data to inform your macro strategy:

  • High-usage macros deserve the most attention during reviews. They are also candidates for optimization since even small improvements compound across thousands of uses.
  • Low-usage macros might be poorly named (agents cannot find them), obsolete, or too specific. Investigate and either improve, merge, or retire them.
  • Agent-specific patterns can reveal training opportunities. If one agent never uses macros, they might need coaching. If another agent has created 50 personal macros, their templates might be worth promoting to shared macros.
